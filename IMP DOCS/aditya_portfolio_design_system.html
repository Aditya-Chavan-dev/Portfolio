Typography

Cormorant Garamond (editorial authority) + JetBrains Mono (technical credibility). The tension between them is the brand identity. Two fonts, zero compromises.

Line Height System
Contextline-height
Display / Headings1.0 – 1.1 — tight, command-like
Dialogue lines1.5 – 1.6 — breathing but controlled
Body copy1.8 — generous, legible at small mono sizes
UI labels / tags1.0 – 1.2 — compact
Form helper text1.6 — room for error messages
Testimonial body1.75 — warm, readable at card scale
Text Rendering & Advanced Rules
-webkit-font-smoothing: antialiased on all text. -moz-osx-font-smoothing: grayscale.
font-feature-settings: "liga" 1, "calt" 1 — enables Cormorant ligatures.
JetBrains Mono: font-feature-settings: "zero" 1 — slashed zero for code-style clarity.
text-rendering: optimizeLegibility on headings only. NOT on body — causes GPU thrash.
Numeric data in tables: font-variant-numeric: tabular-nums — columns align vertically.
Large stat numbers: font-variant-numeric: proportional-nums — feels editorial.
word-spacing: 0 — JetBrains Mono has built-in appropriate spacing.
orphans: 2; widows: 2 on body copy — prevents single words on a line.
hyphens: auto on testimonial body text only. NOT on headings, labels, or UI text.
Text alignment: left-aligned always. Center alignment is never used. Exception: Journey scene numbers (decorative).
Text Decoration & Selection
Underlines: custom, never browser default. 1px accent color, drawn from left via CSS transition (width 0→100%).
Strikethrough: only in Journey scenes for contrast effect. Never elsewhere.
::selection { background: rgba(201,169,110,0.2); color: inherit; } — applied globally.
caret-color: var(--color-accent) on all text inputs and textareas.
::placeholder { color: var(--color-text-muted); font-size: 12px; font-weight: 300; }

Layout System

Left-aligned, document-like. Content never stretches to viewport edges. Negative space is structural. Max-width: 1100px.

Page & Component Containers

Page wrappermax-width: 1100px, mx-auto, px-16 (desktop), px-6 (mobile)
Welcome screenFull viewport (100vw × 100vh). Content anchored top-left: padding 64px desktop, 24px mobile. No other elements.
Hub pagemax-width: 1100px. Two-column desktop (left 60%, right 40%). Single column mobile.
Section pagesmax-width: 900px. Single column. Content left-aligned. Never centered.
Journey scenesFull viewport. Content centered vertically (only exception to left-align — cinematic intent).
Project detail modalmax-width: 680px, centered overlay. Mobile: full-screen bottom sheet.
Testimonial pagemax-width: 560px. Single column, left-aligned.
Admin panelmax-width: 1200px. Sidebar + main content.

Testimonial strip (mobile)Horizontal flex. CSS animation scrolls rightward. overflow: hidden.

Layout Rules
All text is left-aligned. Center alignment is never used. The portfolio is a document, not a banner ad.
Negative space is intentional. Sections breathe. Crowding is a quality-signal failure.
Content max-width: 1100px for layout. 540px for body text paragraphs.
Cards per row: max 2 for project cards, max 2 for Quick Access cards. Never 3+.
Sticky elements: nav only. No sticky cards, no sticky panels.

The 10 Laws
01Every color, size, spacing and duration references a token. Never a hardcoded value.
02The accent color appears sparingly. If it is everywhere, it means nothing.
03Everything is left-aligned. Center alignment is generic.
04Negative space is structural. Filling space is a design failure.
05Dark mode is the primary experience. Light mode is equally polished.
06Font sizes never drop below the clamp minimum. The mobile floor is sacred.
07Animations use only transform and opacity. Never animate layout properties.
08once: true — elements animate in once, never on scroll-up. Maturity signals.
09One primary button per visual section. Its scarcity is its power.
10All interactive elements have all five states: rest, hover, active, focus, disabled.
